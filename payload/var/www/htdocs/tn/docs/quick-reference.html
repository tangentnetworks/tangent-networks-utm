Foreword

This document is for anyone (operator, sysadmin, or developer) who needs to find something quickly. Open it first when something is broken, when you cannot remember a path, when you need a command, or when you want to understand what is running and where. It is a map, not an explanation. Every path here is exact. Every command has been verified on the live system. For the reasoning behind any of it, follow the cross-references to the relevant document.

MFS Memory Filesystems

Critical: These two paths are mounted as swap-backed memory filesystems. Their contents are destroyed on every reboot, including WebUI-initiated reboots and power cycles. This is intentional and correct. rc.local recreates the necessary files at boot. If these paths appear as regular directories with no mount, check /etc/fstab -- the MFS entries may be missing.

/var/www/htdocs/tn/data/logs/pf/
128MB MFS. Contains pflog1.log, a live tcpdump feed of all traffic logged by PF. newsyslog truncates at 64MB and calls pf-tcpdump.sh to archive before truncation. Empty after every reboot; this is correct.
/var/www/htdocs/tn/data/pipes/pmacct/
64MB MFS. Contains ext_if_json.log and int_if_json.log, the live pmacct network flow feeds. newsyslog truncates each at ~16MB and sends SIGHUP to pmacctd. Empty after every reboot; this is correct.

Interfaces

%%EXT_IF%% -- WAN
192.168.122.92, fd00:cafe:1:0:8bce:e1c1:a74:d3b1. External interface. Default gateway: %%WAN_GW4%% (see /etc/mygate) and IPv6 gateway %%WAN_GW6%% (see /etc/mygate6).
%%INT_IF%% -- LAN
%%INT_IP4%%/24, %%INT_IP6%%. Internal interface. DHCP served on this interface. LAN clients receive IPv4 leases from dhcpd and IPv6 via SLAAC (rad).
pflog0
PF block log interface. pflogd writes to /var/log/pflog (standard binary pcap). Contains blocked packets only.
pflog1
PF full traffic log interface. All traffic logged via match log (to pflog1) on { %%EXT_IF%% %%INT_IF%% } in pf.conf. tcpdump reads this interface and writes to the MFS path above. Brought up by /etc/hostname.pflog1, which must contain: up.

pmacct Three-Instance Architecture

pmacct runs as three separate pmacctd processes. Knowing which instance does what is essential for diagnosing flow accounting gaps.

ext_if_json_mfs -- external interface MFS writer
Config: /etc/pmacct/ext_if_json_mfs.conf. Output: data/pipes/pmacct/ext_if_json.log (MFS, lost on reboot). Starts immediately at boot. newsyslog sends SIGHUP at ~16MB. PID: data/run/pmacct/pmacct-ext_if_json_mfs.pid.
int_if_json_mfs -- internal interface MFS writer
Config: /etc/pmacct/int_if_json_mfs.conf. Output: data/pipes/pmacct/int_if_json.log (MFS, lost on reboot). Starts immediately at boot. newsyslog sends SIGHUP at ~16MB. PID: data/run/pmacct/pmacct-int_if_json_mfs.pid.
ext_if_json_log -- disk archive writer
Config: /etc/pmacct/ext_if_json_log.conf. Output: data/network/pmacct/ext/ext_if-%Y%m%d-%H%M.json (quarter-hour files on disk, persistent). Starts at the next quarter-hour boundary after boot, up to 15 minutes after startup. An empty PID file immediately after boot is correct. rc.local merges yesterday's files into YYYY-MM-DD.json at boot and purges files older than 7 days. PID: data/run/pmacct/pmacct-ext_if_json_log.pid.

PF Divert Chain

PF intercepts LAN traffic on the following ports and diverts it to the corresponding service listening on loopback. After inspection, traffic is re-injected and may pass through Snort IPS before forwarding.

LAN port 21 (FTP)
Diverted to ftp-proxy on lo:8021 / lo:8022.
LAN port 80 (HTTP)
Diverted to SSLproxy on lo:8081 for transparent interception.
LAN port 443 (HTTPS)
Diverted to SSLproxy on lo:8443 for transparent TLS interception.
LAN port 110 (POP3)
Diverted to SSLproxy on lo:8994, forwarded to p3scan for mail scanning.
LAN port 995 (POP3S)
Diverted to SSLproxy on lo:8995, forwarded to p3scan for encrypted mail scanning.
LAN port 993 (IMAPS)
Diverted to SSLproxy on lo:8993, forwarded to p3scan for encrypted IMAP mail scanning. IPv4 and IPv6.
LAN port 25 (SMTP)
Diverted to smtp-gated on lo:8464.
LAN port 465 (SMTPS)
Diverted to smtp-gated on lo:8465.
LAN port 587 (submission)
Diverted to smtp-gated on lo:8466.
LAN IM ports 1863, 5190, 5050, 6667
Diverted to imspector on lo:16667.
Loopback ports 8080, 8110, 9199
Diverted to Snort IPS via divert-packet port 700. These are the post-proxy re-injection ports where traffic arrives after SSLproxy has decrypted it.

Inspection Chain

Encrypted traffic from LAN clients passes through two inspection stages. First, PF diverts it to SSLproxy, which decrypts the TLS session, makes the plaintext available for inspection, and re-encrypts before forwarding. The decrypted traffic is then re-injected on loopback, where PF diverts it a second time to Snort IPS running in divert-packet mode on port 700. Snort inspects the plaintext against IPS rules and either drops the packet or allows it through to the destination. A block at either stage drops the connection: SSLproxy blocks before Snort ever sees the traffic, and Snort blocks after decryption but before anything reaches the destination.

Key System Files

/etc/pf.conf
PF ruleset. Addons anchor loaded from /etc/pf/pf-addons.conf.
/etc/sysctl.conf
Kernel tuning. net.bpf.bufsize=2097152 is critical: if lost after sysupgrade, pflog and pmacct will silently drop packets.
/etc/login.conf
Resource limits. The daemon class sets datasize=8192M and openfiles-max=65536, which are required for e2guardian and ClamAV. If lost after sysupgrade, these daemons will hit resource limits silently.
/etc/syslog.conf
Routes all service output to data/logs/<service>/. Uses !!program (exclusive) and !program (filter) rules. httpd routing does not work; TNWAF.pm writes httpd logs directly.
/etc/newsyslog.conf
Active only for the two MFS paths. All other logs handled by tangent_logrotate.sh.
/etc/fstab
Contains the two MFS mount entries. If missing, the MFS paths become regular disk directories with no size limit.
/etc/rc.conf.local
resolvd_flags=NO (unbound handles all DNS). dhcpd_flags=%%INT_IF%%.
/etc/hostname.pflog1
Must contain up. If missing, pflog1 never comes up and all traffic logging goes dark.
/var/db/dhcpd.leases
DHCP lease ground truth. dhcpd does not produce a traditional log; this file is the authoritative source for lease activity.
/var/db/snortsentry.state
snortsentry block escalation state. Persists across reboots. To reset: stop snortsentry, delete this file, then restart snortsentry.
/usr/local/share/tangent/rc.backup.*
Timestamped backups of /etc/rc created automatically by tangent_logrotate.sh when it repairs the rotation hook after a sysupgrade.

Common Recovery Commands

sh 
# Check boot log -- first stop when dashboard is dark after reboot
tail -50 /var/www/htdocs/tn/data/logs/bootlog/rc.local.log

# Check if a WebUI runner is alive
cat /var/www/htdocs/tn/data/run/webui/queue_runner.pid
kill -0 $(cat /var/www/htdocs/tn/data/run/webui/queue_runner.pid)

# Restart a stopped WebUI runner manually
/usr/local/sbin/queue_processor_runner.sh &

# Check all WebUI runner PID files
for f in /var/www/htdocs/tn/data/run/webui/*.pid; do
    pid=$(cat "$f" 2>/dev/null)
    name=$(basename "$f" .pid)
    if [ -n "$pid" ] && kill -0 "$pid" 2>/dev/null; then
        echo "OK   $name ($pid)"
    else
        echo "DEAD $name"
    fi
done

# Check snort_block table
pfctl -t snort_block -T show

# Reset PF addons anchor (clears all user-added rules)
pfctl -a addons -F all

# Reload PF ruleset
pfctl -f /etc/pf.conf

# Check MFS mounts are present
mount | grep mfs

# Check MFS space usage
df -h /var/www/htdocs/tn/data/logs/pf \
       /var/www/htdocs/tn/data/pipes/pmacct

# Manually rotate logs now
/usr/local/sbin/tangent_logrotate.sh

# Truncate MFS logs manually if full
truncate -s0 /var/www/htdocs/tn/data/logs/pf/pflog1.log
truncate -s0 /var/www/htdocs/tn/data/pipes/pmacct/ext_if_json.log
truncate -s0 /var/www/htdocs/tn/data/pipes/pmacct/int_if_json.log

# Reset snortsentry block escalation state
rcctl stop snortsentry
rm /var/db/snortsentry.state
rcctl start snortsentry

# Check collectd socket exists
ls -la /var/www/htdocs/tn/data/sockets/collectd/collectd.sock

# Check ClamAV socket exists
ls -la /var/www/htdocs/tn/data/tmp/clamav/clamd.socket

# Check sysctl BPF buffer (critical for pflog + pmacct)
sysctl net.bpf.bufsize net.bpf.maxbufsize

# Verify login.conf daemon class limits
grep -A10 '^daemon:' /etc/login.conf

Known Benign Conditions

pflog1.log is empty after reboot
MFS is fresh. tcpdump is writing to it. Normal.
ext_if_json.log and int_if_json.log are empty after reboot
MFS is fresh. pmacctd is writing to them. Normal.
pmacct-ext_if_json_log.pid is empty for up to 15 minutes after boot
That instance waits for the next quarter-hour boundary. Normal.
process_monitor.pl has not run immediately after boot
Delayed to the 15-minute boundary plus one minute. Normal.
Boot appears to pause for up to 90 seconds at ClamAV
rc.local is waiting for clamd.socket. Normal on cold hardware; not a hang.
pf_asn_lookip.pid and pf_asn_lookup.pid contain the same PID
Cosmetic duplicate from a name and pidfile mismatch in start_service. One process, two PID files. No operational impact.
data/logs/collectd/collectd.log is 166MB or larger
collectd is very chatty. Normal. tangent_logrotate.sh rotates it daily.

Summary

  • Everything Tangent Networks owns lives under /var/www/htdocs/tn/data/: one root, fully inspectable with standard Unix tools.
  • Two paths are on MFS memory filesystems and are empty after every reboot: data/logs/pf/ (128MB) and data/pipes/pmacct/ (64MB). This is correct behaviour.
  • If the dashboard is dark after reboot, read data/logs/bootlog/rc.local.log first.
  • pmacct runs three instances: two MFS writers that start immediately, and one disk writer that starts at the next quarter-hour boundary.
  • The BPF buffer (net.bpf.bufsize=2097152) and the login.conf daemon class are silent hard dependencies. If either is lost after sysupgrade, services will fail without any obvious error.
